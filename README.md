# Wasselny — تطبيق نقل ركاب (صيدا) — V1

نسخة أولى مجانية بالكامل، مبنية على القرارات التي اتفقنا عليها: نسخة ويب فقط، نطاق صيدا، بدون أي خدمة مدفوعة.

## 1. ماذا يحتوي المشروع

- **تطبيق الراكب** (`public/rider.html`): تسجيل/دخول، طلب رحلة من سائق قريب على الخريطة، تتبع حالة الطلب، تقديم شكوى.
- **تطبيق السائق** (`public/driver.html`): تسجيل (مع رفع صورة الهوية ودفتر السير)، دخول، تفعيل/إيقاف الاستعداد، استقبال الطلبات وقبولها/رفضها، إنهاء الرحلة وتسجيل السعر.
- **لوحة الإدارة** (`public/admin.html`): مراجعة وتحقق السائقين (مع حذف الصور تلقائياً فور التحقق)، تعطيل/تفعيل الحسابات، عرض الرحلات وتعديل الأسعار، عرض الشكاوى ومعالجتها، إحصائيات عامة.
- **الخادم** (`server.js` + `routes/`): Node.js + Express.
- **قاعدة البيانات**: ملف JSON محلي (`data/db.json`) عبر مكتبة lowdb — مناسب لحجم صغير (~50 سائقاً). لا حاجة لقاعدة بيانات منفصلة في هذه المرحلة.

## 2. التشغيل على جهازك (للتجربة)

يتطلب تثبيت [Node.js](https://nodejs.org) (نسخة 18 أو أحدث) على جهازك أولاً.

```bash
cd waslni
npm install
npm start
```

ثم افتح المتصفح على: `http://localhost:3000`

- صفحة الراكب: `http://localhost:3000/rider.html`
- صفحة السائق: `http://localhost:3000/driver.html`
- صفحة الإدارة: `http://localhost:3000/admin.html`

## 3. بيانات دخول الإدارة الافتراضية

```
اسم المستخدم: admin
كلمة السر: admin123
```

**يجب تغييرها فوراً قبل أي استخدام فعلي.** غيّرها عبر متغيرات البيئة (لا تُكتب كلمة السر داخل الكود نفسه):

```bash
ADMIN_USER=اسم_جديد ADMIN_PASS=كلمة_سر_قوية JWT_SECRET=نص_عشوائي_طويل npm start
```

`JWT_SECRET` هو مفتاح توقيع جلسات الدخول — غيّره لأي نص عشوائي طويل قبل الاستخدام الفعلي (وإلا يمكن لأي شخص يعرف القيمة الافتراضية تزييف جلسة دخول).

## 4. النشر المجاني على الإنترنت (Render)

لتشغيل التطبيق بشكل دائم بدون أن يبقى جهازك مفتوحاً، استخدم خدمة استضافة مجانية مثل Render:

1. أنشئ حساباً مجانياً على [render.com](https://render.com).
2. ارفع هذا المشروع إلى مستودع GitHub (يمكن أن يساعدك أي مطور بهذه الخطوة لو لم تكن معتاداً عليها).
3. من Render: New → Web Service → اختر المستودع.
4. الإعدادات:
   - Build Command: `npm install`
   - Start Command: `npm start`
   - الخطة (Plan): Free
5. أضف متغيرات البيئة من تبويب Environment: `ADMIN_USER`, `ADMIN_PASS`, `JWT_SECRET` بالقيم الجديدة.
6. بعد النشر، ستحصل على رابط عام (مثلاً `https://waslni.onrender.com`) شاركه مع السائقين والركاب.

**تنبيه مهم:** الطبقة المجانية من Render "تنام" بعد فترة من عدم الاستخدام، وأول طلب بعدها يكون أبطأ (عدة ثوانٍ). هذا مقبول لمرحلة الاختبار، ويمكن إعادة النظر فيه عند نمو الاستخدام.

**تنبيه آخر بخصوص قاعدة البيانات على Render المجاني:** التخزين على الطبقة المجانية غير دائم بشكل مضمون عند إعادة تشغيل الخدمة (قد يُعاد ملف `data/db.json` للحالة الافتراضية). لتشغيل تجريبي هذا مقبول. عند الانتقال لاستخدام فعلي مستمر، يُنصح بنقل قاعدة البيانات لاحقاً إلى خدمة تخزين دائم (مثل قرص Render المدفوع بسعر رمزي، أو الانتقال إلى Postgres عبر Supabase).

## 5. كيف يعمل التحقق من هوية السائق

1. السائق يسجّل ويرفع صورة الهوية ودفتر السير.
2. الحساب يبقى "قيد المراجعة" — لا يمكنه تسجيل الدخول.
3. الإدارة تفتح لوحة التحكم، تراجع الصورتين، وتضغط "تأكيد التحقق".
4. عند التأكيد: الصورتان تُحذفان فوراً من القرص، ويبقى فقط رقم الهوية ورقم الرخصة وحالة "تم التحقق" + اسم من راجع + التاريخ.
5. بعد ذلك يمكن للسائق تسجيل الدخول.

## 6. نقاط مفتوحة (حسب وثيقة النطاق السابقة)

- آلية تسوية العمولة نقدية يدوية بين السائق والإدارة — لا يوجد كود لها لأنها عملية تتم خارج التطبيق.
- التطبيق الفعلي على المتاجر (Android/iOS) مؤجل لحين توفر ميزانية.
- لا تتبع GPS حي أثناء الرحلة — فقط لحظة اختيار السائق الأقرب.

## 7. حدود معروفة في هذه النسخة (طبيعية لمرحلة V1)

- قاعدة بيانات JSON بسيطة — مناسبة لعشرات السائقين، تحتاج ترقية إذا كبر المشروع كثيراً.
- لا يوجد تحقق SMS OTP — التسجيل برقم هاتف وكلمة سر فقط، والثقة تأتي من المراجعة اليدوية للسائقين.
- موقع السائق يُرسل فقط أثناء حالة "متوفر" — وهذا يتطلب أن يبقي السائق المتصفح مفتوحاً وفي المقدمة، كما اتفقنا.
